# NGINX Configuration Analysis (Variant 3)

## Описание задачи

Дан конфигурационный файл NGINX для балансировки трёх backend-сервисов. Пользователи жалуются на:

* потерю сессий при авторизации;
* ошибки 502 при падении одного backend;
* медленную загрузку статических файлов.

Задача — выявить проблемы в конфигурации, объяснить последствия и предложить исправления.

---

## Проблема 1. Отсутствие sticky sessions

**Что не так:**
В блоке `upstream` не настроено закрепление клиента за конкретным backend.

**Почему это плохо:**
Если сессии хранятся в памяти приложения, пользователь может попасть на другой backend и “потерять” авторизацию.

**Как исправить:**

```nginx
upstream api {
    ip_hash;
    server backend1:8080;
    server backend2:8080;
    server backend3:8080;
}
```

---

## Проблема 2. Нет параметров отказоустойчивости

**Что не так:**
Не заданы `max_fails` и `fail_timeout`.

**Почему это плохо:**
NGINX продолжает слать запросы на упавший backend → пользователи получают 502.

**Как исправить:**

```nginx
server backend1:8080 max_fails=3 fail_timeout=10s;
```

---

## Проблема 3. Нет `proxy_next_upstream`

**Что не так:**
Нет повторной отправки запроса на другой backend.

**Почему это плохо:**
При падении одного backend запрос сразу падает с ошибкой.

**Как исправить:**

```nginx
proxy_next_upstream error timeout http_500 http_502 http_503 http_504;
proxy_next_upstream_tries 3;
```

---

## Проблема 4. Нет proxy-заголовков

**Что не так:**
Не передаются `Host`, `X-Real-IP`, `X-Forwarded-For`.

**Почему это плохо:**
Backend не видит реальный IP и хост → возможны проблемы с логированием, редиректами и безопасностью.

**Как исправить:**

```nginx
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
```

---

## Проблема 5. Малое количество соединений

**Что не так:**

```nginx
worker_connections 64;
```

**Почему это плохо:**
Сервер не справляется с нагрузкой → медленные ответы и ошибки.

**Как исправить:**

```nginx
worker_connections 1024;
```

---

## Проблема 6. Один worker-процесс

**Что не так:**

```nginx
worker_processes 1;
```

**Почему это плохо:**
NGINX использует только одно ядро CPU → низкая производительность.

**Как исправить:**

```nginx
worker_processes auto;
```

---

## Проблема 7. Нет кэширования статики

**Что не так:**
Нет `expires` и `Cache-Control`.

**Почему это плохо:**
Браузер не кэширует файлы → медленная загрузка.

**Как исправить:**

```nginx
expires 7d;
add_header Cache-Control "public, max-age=604800";
```

---

## Проблема 8. Неправильное использование root

**Что не так:**

```nginx
root /var/www;
```

**Почему это плохо:**
Можно получить 404, если структура файлов не совпадает.

**Как исправить:**

```nginx
alias /var/www/static/;
```

---

## Проблема 9. Нет sendfile

**Что не так:**
Отсутствует оптимизация передачи файлов.

**Почему это плохо:**
Статика отдаётся медленнее.

**Как исправить:**

```nginx
sendfile on;
```

---

## Проблема 10. Нет таймаутов

**Что не так:**
Не заданы proxy timeout’ы.

**Почему это плохо:**
Подвисшие запросы блокируют соединения.

**Как исправить:**

```nginx
proxy_connect_timeout 2s;
proxy_read_timeout 5s;
proxy_send_timeout 5s;
```

---

## Итог

Исходный конфиг не учитывает отказоустойчивость, масштабируемость и производительность.
Исправления позволяют:

* избежать потери сессий;
* корректно обрабатывать падение backend;
* ускорить загрузку статики;
* повысить общую стабильность системы.
